** Interface
-> 코드 내 계약 (약속, 규칙)을 정의하는 방법
-> 일반적으로 변수, 함수, 클래스의 타입을 체크하기 위해 사용

** 추상 클래스 vs 인터페이스
-> 추상 클래스는 추상 메소드가 포함된 클래스로 일반 메소드를 포함할 수 있음
-> 인터페이스는 모든 메소드가 추상 메소드여야함

** 추상 클래스
-> 상속을 통해 추상 메소드의 구현을 강제하여 자식 클래스에서 일부 동작을 구현하도록 만든 것

** Properties
-> 프로퍼티 선언 시 이름 끝에 ? 를 붙여 인터페이싀에 속하지 않는 프로퍼티의 사용을 방지
-> readonly 프로퍼티 : 수정이 불가하도록 설정
